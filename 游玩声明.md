# 《影域博弈》详细游玩声明

## 一、项目定位

《影域博弈》是一款基于 C++ 开发的 2D 双人联机 PVP 游戏。游戏采用非对称对抗设计，一名玩家扮演“光刃使”，另一名玩家扮演“影行者”。双方围绕地图中的光源、阴影、中心据点和生命值进行对抗。

本项目的目标不是做一个只有移动和射击的简单小程序，而是做一个可以完整演示的课程项目。它包含开始菜单、房间建立、加入房间、联机同步、角色技能、生命值、得分、占点进度、胜负结算和离线演示模式。

## 二、启动方式

最推荐的启动方式是双击仓库根目录中的：

```text
启动游戏.bat
```

这个脚本会检查是否存在游戏程序。如果主程序已存在，它会直接启动游戏；如果不存在，它会尝试调用 `mingw32-make` 自动编译。

如果你想在命令行中启动，也可以运行中文启动器：

```text
启动游戏.bat
```

桌面上如果已经创建了“影域博弈”启动器，也可以直接双击桌面启动器进入游戏。

## 三、游戏模式说明

游戏当前提供三种进入方式。

### 1. 建立房间

在开始菜单按 `H`，当前电脑会成为主机。主机玩家操控“光刃使”。主机会监听 `54000` 端口，等待另一名玩家加入。

主机建立房间后，如果还没有客户端加入，游戏画面会显示等待对手连接。客户端成功连接后，对局自动开始。

### 2. 加入房间

在开始菜单输入主机 IP 后按 `J`，当前电脑会作为客户端加入房间。客户端玩家操控“影行者”。

如果是在同一台电脑双开测试，IP 保持默认：

```text
127.0.0.1
```

如果是两台电脑局域网联机，客户端需要输入主机电脑的局域网 IP，例如：

```text
192.168.1.23
```

### 3. 本机离线双人演示

在开始菜单按 `O`，进入本机离线演示。这个模式不需要网络，可以在一台电脑上同时控制两个角色，适合测试玩法、课堂演示或网络环境不稳定时备用。

## 四、基础操作说明

### 光刃使

光刃使是偏正面压制的角色。光刃使在亮处更强，适合围绕光源作战。

操作键位：

- `W`：向上移动。
- `A`：向左移动。
- `S`：向下移动。
- `D`：向右移动。
- `F`：普通攻击。
- `G`：强光爆发。
- `Q`：放置光源。
- `R`：重新开始对局。

技能说明：

- 普通攻击会发射光弹，命中影行者会造成伤害。
- 强光爆发会在自身附近制造一次范围压制，适合在近距离逼退影行者。
- 放置光源会在当前位置生成光照区域，改变地图局势。

战斗建议：

- 尽量围绕光源移动，不要长时间离开亮区。
- 影行者冲刺靠近时，可以使用强光爆发反制。
- 中心据点附近放置光源，可以增强占点能力。

### 影行者

影行者是偏游走、突袭和反制的角色。影行者在暗处更灵活，适合避开光源寻找切入机会。

联机客户端操作键位：

- `W`：向上移动。
- `A`：向左移动。
- `S`：向下移动。
- `D`：向右移动。
- `F`：普通攻击。
- `G`：暗影冲刺。
- `Q`：释放影域。
- `R`：重新开始对局。

本机离线模式下，影行者使用另一套键位：

- 方向键：移动。
- `Enter`：普通攻击。
- `K` 或 `Shift`：暗影冲刺。
- `L` 或 `Ctrl`：释放影域。

技能说明：

- 普通攻击会发射暗影弹，命中光刃使会造成伤害。
- 暗影冲刺可以快速位移，接近敌人或脱离危险区域。
- 影域可以削弱附近光源，让局部区域重新偏向黑暗。

战斗建议：

- 不要正面硬顶光刃使的光源阵地。
- 利用暗处移动速度优势绕侧面。
- 等光刃使技能进入冷却后再冲刺切入。
- 用影域削弱中心据点附近的光源，再尝试占点。

## 五、胜负规则

游戏采用得分胜利规则。任意一方先获得 3 分，即可赢得本局。

得分方式包括：

- 击败对方角色。
- 控制中心据点并累计占点进度。

当玩家生命值降为 0 时，该玩家会被判定为被击败，并在起始区域复活。击败者获得分数。

中心据点位于地图中央。只有一方站在中心区域时，占点进度会向该方推进。如果双方都在据点内，或者双方都不在据点内，占点进度不会快速变化。当占点进度达到阈值时，占点方获得分数，进度重置。

## 六、画面元素说明

顶部信息栏显示：

- 双方比分。
- 当前占点进度。
- 对局持续时间。
- 主机/客户端/离线状态。

底部信息栏显示：

- 光刃使攻击、强光、光源技能的冷却时间。
- 影行者攻击、冲刺、影域技能的冷却时间。

地图元素说明：

- 黄色圆形区域表示光源范围。
- 蓝色角色表示影行者。
- 黄色角色表示光刃使。
- 地图中央大圆是中心据点。
- 飞行的小圆点是双方的普通攻击弹体。
- 大范围闪烁圆形表示强光爆发或影域效果。

## 七、联机同步说明

本游戏使用 TCP 进行基础联机同步。主机负责运行主要游戏逻辑，客户端发送自己的输入。主机将完整世界状态同步给客户端。

同步内容包括：

- 玩家位置。
- 玩家生命值。
- 玩家技能冷却。
- 弹体位置。
- 光源状态。
- 比分。
- 占点进度。
- 游戏阶段。
- 胜负结算。

这意味着客户端不是各自独立计算一套世界，而是跟随主机同步状态。这样可以减少双方画面不一致的情况，也更适合课程演示。

## 八、同机双开演示步骤

这是最稳定、最适合课堂快速展示的方式。

1. 双击 `启动游戏.bat` 打开第一个游戏窗口。
2. 第一个窗口按 `H` 建立房间。
3. 再双击 `启动游戏.bat` 打开第二个游戏窗口。
4. 第二个窗口确认 IP 为 `127.0.0.1`。
5. 第二个窗口按 `J` 加入房间。
6. 第一个窗口操控光刃使，第二个窗口操控影行者。
7. 双方移动、攻击、释放技能，观察两个窗口中的状态是否同步。

## 九、两台电脑联机演示步骤

1. 确认两台电脑处于同一局域网。
2. 在主机电脑上打开游戏，按 `H`。
3. 查找主机电脑 IP。可以在 PowerShell 中输入：

```powershell
ipconfig
```

4. 在客户端电脑打开游戏。
5. 在客户端开始菜单中输入主机 IP。
6. 客户端按 `J` 加入房间。
7. 如果无法连接，检查防火墙是否允许游戏联网，或者临时允许本项目生成的游戏主程序通过防火墙。

## 十、常见问题

### 1. 双击启动器没有反应

请进入仓库目录，确认是否存在：

```text
build\YingYuBoYi.exe
```

如果不存在，说明还没有编译成功。可以在仓库根目录运行：

```powershell
mingw32-make
```

### 2. 客户端连不上主机

请检查以下几项：

- 主机是否已经按 `H` 建房。
- 客户端输入的 IP 是否正确。
- 两台电脑是否在同一局域网。
- Windows 防火墙是否拦截程序。
- 端口 `54000` 是否被占用。

### 3. 中文显示异常

游戏窗口使用 Unicode 绘制中文。如果仍出现显示异常，请确认系统中存在“微软雅黑”或“Microsoft YaHei UI”等常见中文字体。

### 4. 想一个人测试两个角色

在菜单按 `O` 进入本机离线演示即可。该模式不需要网络。

## 十一、课堂展示建议

展示时可以按下面顺序讲解：

1. 先打开菜单，说明项目是 C++ 2D 联机 PVP。
2. 进入本机离线模式，快速展示角色移动、攻击和技能。
3. 双开窗口演示联机，让老师看到两个客户端同步。
4. 展示光源和影域的对抗关系，说明不是普通射击游戏。
5. 展示中心占点、比分和胜负结算。
6. 说明 AI 参与了需求拆解、玩法设计、代码结构、联网同步和测试说明整理。

可以强调的项目亮点：

- 非对称角色设计。
- 光影机制影响战局。
- 支持联机同步。
- 有完整 UI 状态反馈。
- 有离线演示兜底。
- 项目可运行、可展示、可继续扩展。

## 十二、当前版本边界

当前版本是基础可玩版，重点是完整实现课程要求中的基础功能。它还不是商业级游戏，也没有复杂匹配系统、账号系统、专业美术资源或高精度网络预测。

后续可以继续扩展：

- 加入更精美的角色贴图和技能特效。
- 加入音效和背景音乐。
- 增加地图障碍物。
- 加入更多胜负模式。
- 增加网络断线重连提示。
- 增加更完整的房间界面。
- 加入 AI 生成的角色背景、技能说明和美术素材。

## 十三、游玩声明

本游戏用于课程项目展示和学习交流。游戏中的联机功能面向局域网或本机双开测试，不建议直接暴露到公网环境。运行游戏前请确认程序来源可信，并在需要联机时允许 Windows 防火墙放行该程序。

如果在课堂展示中网络环境不稳定，可以优先使用本机离线双人演示模式，保证项目功能能够稳定呈现。
- Windows 防火墙是否拦截程序。
- 端口 `54000` 是否被占用。
